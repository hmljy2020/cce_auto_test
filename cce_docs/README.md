# CCE Instruction Documents

This directory stores the Agent-maintained understanding for CCE instructions.

## Metadata Format

Each instruction may have one metadata JSON file. The metadata is intentionally
minimal: humans only maintain the instruction name, metadata version, and the
selected overload signatures with approval status.

Example:

```json
{
  "instruction": "copy_gm_to_ubuf",
  "metadata_version": 1,
  "overloads": [
    {
      "status": "approved_sweep",
      "signature": "void copy_gm_to_ubuf(__ubuf__ void* dst, __gm__ void* src, uint8_t sid, uint16_t nBurst, uint16_t lenBurst, uint16_t srcStride, uint16_t dstStride);"
    }
  ]
}
```

If an instruction has multiple overloads, record each overload as one entry in
`overloads`:

```json
{
  "instruction": "vsub",
  "metadata_version": 1,
  "overloads": [
    {
      "status": "approved_sweep",
      "signature": "void vsub(__ubuf__ half* dst, __ubuf__ half* src0, __ubuf__ half* src1, uint64_t config);"
    },
    {
      "status": "approved_smoke",
      "signature": "void vsub(__ubuf__ half* dst, __ubuf__ half* src0, __ubuf__ half* src1, uint8_t repeat, uint8_t dstBlockStride, uint8_t src0BlockStride, uint8_t src1BlockStride, uint8_t dstRepeatStride, uint8_t src0RepeatStride, uint8_t src1RepeatStride);"
    },
    {
      "status": "draft",
      "signature": "void vsub(__ubuf__ float* dst, __ubuf__ float* src0, __ubuf__ float* src1, uint64_t config);"
    }
  ]
}
```

For namespaced functions, use the fully qualified name in `signature`:

```json
{
  "status": "approved_sweep",
  "signature": "void bisheng::cce::vsub(__ubuf__ half* dst, __ubuf__ half* src0, __ubuf__ half* src1, uint64_t config);"
}
```

## Signature Canonicalization

`signature` must be a canonical signature string. It should be generated by a
script from the raw declaration in `stub_fun.h`, not manually reformatted in a
different style.

The canonicalization flow is:

```text
stub_fun.h raw declaration
  -> parse and canonicalize
  -> metadata.overloads[].signature
```

Canonical signatures must be:

- one line.
- terminated by `;`.
- equivalent to the selected `stub_fun.h` overload.
- stable in whitespace, pointer, reference, namespace, type, and parameter-name
  formatting according to the script's canonicalization rules.

Database rows should copy `metadata.overloads[].signature` exactly.

## Status Values

The allowed status values are:

- `draft`: recorded but not confirmed; not allowed for smoke or formal sweep.
- `approved_smoke`: allowed for temporary smoke validation only.
- `approved_sweep`: allowed for formal micro-benchmarking and database writes.

Do not write `stale` into metadata. Staleness is a runtime check result produced
by comparing the metadata signature with the current `stub_fun.h`.

## Derived Fields

Do not hand-write derived fields such as `signature_id`, parameter lists, return
type, or signature hash. Scripts should derive them from `signature`.

The stable signature identity is derived from:

```text
namespace + function name + return type + ordered parameter types
```

Parameter names are kept in the signature for readability and for later case
generation, but they do not define the signature identity.
